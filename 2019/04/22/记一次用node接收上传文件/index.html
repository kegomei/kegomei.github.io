记一次用node接收上传文件

node接收文件上传

在做课程设计的时候有一个注册时上传用户头像的需求，先来说说自己的思路：

  1. 首先，前端上传图片，发送请求至后端
  2. 后端将文件接收，修改文件名，存入资源，然后将文件路径加入到用户信息的数据库中
  3. 服务端将文件url返回给前端，供用户预览
  4. 用户在使用页面时会直接使用数据库中该用户对应的头像url，显示出来
  5. 用户可以重新上传，重新上传后，更新文件，其他不变

后端

express提供了一个插件multiparty可以完成后端的需求，目前仅了解到其处理文件方面的功能

前端

前端在后台图片更新后前端图片并不会自动更新，因为图片路径没有变，所以还是从缓存中加载图片

解决办法是在图片路径后面加上一个随机数作为请求参数，由于该参数是无意义的，所以不影响

我用的是vue框架，因此前端要做的就是给路径后加一个”?t=tmp”

在图片更换请求的回调中改变tmp就行了
